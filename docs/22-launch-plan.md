# 22 — Launch Plan

## Pre-Launch (T-30 to T-7)

- [ ] Freeze MVP scope (Phases 1–14).
- [ ] Pen test on staging.
- [ ] Load test critical endpoints.
- [ ] Legal review: ToS, Privacy Policy, Cookie Policy.
- [ ] Onboard seed workers (50–100) in launch city.
- [ ] Customer outreach: waitlist via landing page.
- [ ] Support playbook drafted.
- [ ] Status page live.

## Launch Week (T-0)

- [ ] Final deploy to production.
- [ ] Smoke tests across all 3 apps.
- [ ] DNS cut over.
- [ ] Monitoring active.
- [ ] On-call rotation set.
- [ ] Email invites to waitlist.

## T+1 to T+30

- Daily metrics review: signups, jobs posted, applications, completion rate.
- Weekly retros on friction points.
- Iterate on UX issues.
- Activate targeted onboarding emails.
- Ramp worker supply via field ops + referral incentives.

## KPIs (First 30 Days)

| Metric                        | Target   |
| ----------------------------- | -------- |
| Signups (customer)            | 500      |
| Signups (worker)              | 150      |
| Verified workers              | 100      |
| Jobs posted                   | 200      |
| Jobs completed                | 100      |
| Avg time to first application | < 60 min |
| NPS                           | > 30     |

## Risk Register

| Risk              | Mitigation                         |
| ----------------- | ---------------------------------- |
| Low worker supply | Field ops to onboard in person     |
| Low demand        | Targeted Facebook/Instagram ads    |
| Quality issues    | Admin moderation + report handling |
| Infra outage      | Status page + comms playbook       |

## Go/No-Go Criteria

- All MVP acceptance criteria met.
- No P0/P1 bugs open.
- Monitoring + alerts live.
- Support channels staffed.

## Post-Launch Review

- 2-week and 4-week retro.
- Update [25-future-features.md](25-future-features.md) based on learnings.
