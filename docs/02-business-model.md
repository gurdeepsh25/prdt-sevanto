# 02 — Business Model

## Model Type

**Two-sided marketplace (hyperlocal services).**

## Stakeholders

| Stakeholder        | Role                                     | Pays?                                   | Earns?                           |
| ------------------ | ---------------------------------------- | --------------------------------------- | -------------------------------- |
| Customer           | Posts and hires for jobs                 | Yes (future: service fee or commission) | Receives service                 |
| Worker             | Offers services, applies, completes jobs | No (initially)                          | Earns job value                  |
| Admin              | Moderates, supports, analyzes            | —                                       | —                                |
| Sevanto (Platform) | Operates, markets, trusts                | —                                       | Revenue from future monetization |

## MVP Revenue Model

**Free for both sides.** Focus on adoption and liquidity.

## Future Monetization Streams

| Stream                      | Description                            | Phase       |
| --------------------------- | -------------------------------------- | ----------- |
| Platform Commission         | % of completed job value               | Phase 17    |
| Featured Listings           | Paid boost for workers in search       | Phase 17    |
| Premium Worker Subscription | Verified Pro badge, priority placement | Phase 17    |
| Customer Service Fee        | Convenience / dispute resolution fee   | Phase 17    |
| Ads                         | Sponsored categories / banners         | Post-launch |

## Unit Economics (Target)

| Metric                | Target     |
| --------------------- | ---------- |
| Take rate (long-term) | 8–15%      |
| Customer CAC          | < $5       |
| Worker CAC            | < $3       |
| Payback period        | < 6 months |
| Gross margin          | > 60%      |

## Cost Structure (Indicative)

- Cloud infrastructure (Postgres, hosting, storage, email).
- Third-party services (email, SMS, monitoring, maps API later).
- Customer support / moderation ops.
- Marketing & growth.
- Engineering payroll.

## Strategic Moats

1. **Network density** — more workers → more jobs → more workers.
2. **Trust graph** — verified profiles, ratings, reviews.
3. **Local data** — historical job patterns improve matching.
4. **Brand** — recognizable, neighborhood-embedded.

## Risks

| Risk                   | Mitigation                                           |
| ---------------------- | ---------------------------------------------------- |
| Cold-start problem     | Seed with manual worker onboarding in launch city    |
| Quality drift          | Verified profiles + rating system + admin moderation |
| Worker churn           | Build tools, fast leads, fair matching               |
| Regulatory / labor law | Categorize as marketplace, not employer; clear ToS   |
| Platform abuse         | KYC, dispute system, admin tooling                   |

## Key Partnerships (Future)

- Payment gateways (Stripe/Razorpay).
- KYC providers.
- Background check vendors.
- SMS / email providers.
- Maps providers.
